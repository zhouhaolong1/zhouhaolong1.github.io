给萧萧方涯的面试提醒

相爱的时候需要真诚，争执的时候需要沟通，生气的时候需要冷静，愉快的时候需要分享，指责的时候需要谅解，过日子的时候需要包容。

技术面试

可能医院和IT公司还不一样，再加上小生才疏学浅，恳请 萧萧方涯 选择性的吸收。
面试一般分为技术面试和人事面试，一般都是先技术面试再人事面，面完之后就等通知。如果一周内都没有通知，那说明没有戏了。

对于医院我不了解，不过有些东西应该是相通的吧。
一、为了缓解面试严肃的气氛，会谈谈其他的东西，比如天气，新闻等等。
二、虽然医院和IT公司所属行业完全不同，行业不一样关注点可能不同，不过在技术面试中以下几点要注意的应该是差不多的。

  • 面试官问你完全不知道的东西

千万不要不懂装懂，在行家面前还装，会死得很惨的。不过可以这么回答，这个东西目前我还不太了解，不过以后我会加强对这方面的学习。

  • 问些特别难的问题

其实也不要慌，这个主要用来压你工资的。你不知道别人也不知道的呢。并且如果每个问题都让你很轻松的回答出来，你可能就飘飘然了，漫天要价。。对企业是灾难。不过对于特别难的问题，不能想都不想就说不知道，要尽可能的尝试给出解决方案。让面试官看出你积极面对问题，迎接挑战的工作作风和态度。

  • 对于特别熟悉的知识

可以详实，详尽的谈谈你对这个东西的认知和理解，还可以说说这个问题涉及到相关的东西。总之就是，知道的尽量说，往死里说。
每个公司面试时间一般都是有限制的，对于你知道的东西，说得越久，面试官问的问题数量越少，并且还可以看出你的能力水平。

人事面试

技术面试确实无能为力了，那就弄弄人事方面的东西，希望对你有用，嘿嘿。

面试小提醒

  1. 仪容仪表就不用说了，你应该明白，给个小小的建议，头发最好扎起来，显得更青春，阳光，活泼。
  2. 谈吐诚实中肯，面带微笑，不要有小动作（切记），注意眼神交流，显示出自信。
  3. 事先了解要去面试的公司
  4. 千万不能自己主动提薪资的问题，否则会让人感觉你会把钱放在第一位，工作才是第二位，用人单位需要的是敬业的人，所以工资要放到最后谈。不要急一般都会谈的，如果真的没有谈，那这家公司人事真的不咋地。

常见问题

- 自我介绍

这个可以说是必问的，技术面试官和人事一般都会问，一是为了缓解面试紧张的气氛，二是为了了解你，像你这样一个心地善良有学识有修养有内涵有家教的而且还容易沟通的人应该完全不慌，只要大胆有自信的随意谈谈就好。
不过也要注意以下几点：
    1、介绍内容要与个人简历内容大体一致。
    2、表述方式上尽量口语化。
    3、可以谈谈之前的经历
    4、可以谈谈以后你的梦想什么的
总之：让面试官认为你是一个有能力有思想有素养有梦想的求职者就行了。具体发挥看你的了。

- 你的朋友多吗？

这种看似漫无边际的问题，其实是为了从侧面了解你这个人，是不是一个受人喜欢的人。这个问题之所以要说，是想提醒你，面试也不是上战场，会不用太紧张。

- 谈谈你的兴趣爱好？

这个主要是面试官想知道你是否是在持续学习。千万不能说什么，我的爱好就是看电影，听歌，看韩剧什么的。
始终要记住面试官是看你是否有能力，谈这个问题关心的是：你提升个人能力方面的兴趣有哪些，所以要回答一些关于能力提高，学习方面的兴趣爱好。
比如，我喜欢看技术博客，也喜欢喝着咖啡泡技术论坛，空闲的时候会研究新的技术。

- 谈谈你的缺点？

这个问题对于很多人来说很难回答。如果你回答最大的优点就是没缺点，最大的缺点就是优点太多。这种会死的很惨的呢。平时可以开开玩笑这么说，但是面试就呵呵了。
千万也不要真的说自己有哪些哪些缺点。
比如我会说，我是一个急性子，我一有任务我就会加班加点的干，三天的任务可能就一天做完，这样的话没有注意合理分配时间。不过我会在以后的工作中会注意的。
诸如此类的回答还有很多，你自己想一个，反正记住不要真的说自己缺点，要用缺点变相的来点缀自己。

- 对加班的看法？

可以这样回答，每个公司都有项目多的时候，如果工作需要，我肯定义不容辞，埋头苦干。不过，我也会努力提高我的工作效率来减少不必要的加班。

- 你了解过我们公司吗？

这个必须先了解，如果都不先了解，那面试官会觉得你根本不重视这次面试，对他们公司不感兴趣，也不重视他们公司。即使你是一个有能力的人，呵呵的可能性也会大幅度提升。必须要提前了解。

- 你为什么选择我们公司？

面试官试图从中了解你求职的动机、愿望以及对此项工作的态度。
建议从行业、企业和岗位这三个角度来回答。
可以这样回答：我十分看好贵公司所在的行业，我认为贵公司十分重视人才，而且这项工作很适合我，相信自己一定能做好。

- 你在前一家公司的离职原因是什么？

这个问题目前你不涉及，以后会涉及到，这个问题面试经常问。
不能从上家公司找原因，不能带主观色彩，说上家公司怎么怎么差，如“工作太幸苦”、“人际关系复杂”、“管理太混乱”、“公司不重视人才”、“公司排斥我们某某的员工”等等，貌似全世界都欠你的似的。面试官会对你的印象会很差的。
可以从自身说，比如我在上家公司工作了两年多，有较深的感情，但是我因为我个人原因，我和我的男朋友希望以后在上海发展，因此很遗憾的离开了上一家公司。所以现在重新寻找能发挥我能力的舞台。
要让面试官知道，你是一个有情有义的人。离职原因也可以不必说得太具体，从自身找那些和工作能力无关的原因。千万别说，我完不成任务，被开除了或者被裁员裁掉了，你就呵呵了。

- 想想五年后的你或者未来的职位规划？

对于这个问题你大胆设想，面试官更喜欢有理想，有目标，有激情，有斗志的求职者。

- 你对薪资的要求？

问薪资的话就不要不好意思了，不要怕，可以自己先模拟一下面试官问你这个问题，自己发挥一下。
这个薪资可以参考当地业界的平均水平，根据自己实际情况来综合判定是多少。首先设置一个底线，报价要一般要高于这个底线，说的太低，显示不自信，能力不行，即使能力行，企业也不敢用，可能工作一两个月就因为发现薪资太低而离职了，但是太高了也不好，即使符合了公司的条件，也会因为钱的问题相差太大谈不拢而告吹。
回答方式一：我对工资没有硬性要求。我相信贵公司在处理我的问题上会友善合理。我更注重的是公司给我的发展平台和机会，所以只要条件公平合理，我也不会计较太多
回答方式二：如果你必须自己说出具体数目，不要说一个宽泛的范围，那样你将只能得到最低限度的数字。最好给出一个具体的数字，这样表明你已经对当今的人才市场作了调查，知道像自己这样学历和能力的雇员薪资是多少。

- 你所期望的待遇可能超过了我们公司的预期, 我们无法满足你的要求，你能接受吗？

每个公司都有自己的薪资体系，每个具体的职位都有一个明确的薪资范围。
对于这个问题你可以反问人事，你们给这个职位的薪水范围是多少？一般公司都会给出一个薪水范围的，然后根据实际情况再做回答。如果和期望值相差甚远，可以说让我考虑考虑。

福利待遇

什么是五险一金

所谓“五险”指的是养老保险、医疗保险、失业保险、工伤保险和生育险，而“一金”指的是住房公积金。五险一金具体内容以及作用

五险一金小提醒

每个公司所交基数不一样，知道公司给五险一金交的基数，重点关注养老保险，医疗保险和住房公积金交的百分比。

五险一金具体怎么计算

每个公司基数和所交比例不一样，不过可以知道大致计算方法。

向面试官提问时间

这个反问环节一般都会有，也就几分钟时间，不能不问，也不要问得太多，要直奔主题，不要磨磨唧唧，不问显示你对他们公司或者职位没兴趣，问的太多也不是很好。
这个问题最好自己事先了解他们公司，然后准备一下，当然也可以临场发挥，问问那些你没有听懂的一些东西。
比如可以问未来我在贵公司的发展空间。贵公司未来的发展方向等等。提前想好这个问题，毕竟时间宝贵，所以要珍惜。

总结

面试不是一下就能成功的，即使给你发了offer,也可以多面试几家公司，因为面试就是一个相互选择的过程，对求职者来说肯定是要从自己的福利待遇，未来的发展空间等等因素综合考虑选择哪家公司，被拒绝也不必难过，因为此地不留爷，自有留爷处。

萧萧方涯，祝你好运！

PendingIntent

PendingIntent是什么

PendingIntent就是一个不立即执行的intent，是对intent的封装。
PendingIntent是Android系统管理并持有的用于描述和获取原始数据的对象的标志(引用)。

PendingIntent的作用

PendingIntent是一个延时的Intent，主要用来在某个事件完成后执行特定的Action。PendingIntent包含了Intent及Context，因为PendingIntent是Android系统管理并持有，所以即使Intent所属程序结束，PendingIntent依然有效，可以在其他程序中使用。
PendingIntent一般作为参数传给某个实例，在该实例完成某个操作后自动执行PendingIntent上的Action，也可以通过PendingIntent的send函数手动执行，并可以在send函数中设置OnFinished表示send成功后执行的动作。
PendingIntent通常使用在通知栏、短信、闹钟中。

怎么使用

  1. 获取PendingIntent对象，一般通过下面的静态方法获取

可以理解为，调用这些个静态方法，可以获取能打开或者启动xx的PendingIntent对象,参数主要说一下第四个flags，每个flag有什么含义

  • FLAG_ONE_SHOT: 该 PendingIntent 只作用一次，如果该 PendingIntent 对象已经触发过一次，那么下次再获取该 PendingIntent 并且再触发时，系统将会返回一个 SendIntentException，在使用这个标志的时候一定要注意哦。
  • FLAG_NO_CREATE: 如果当前系统中不存在相同的 PendingIntent 对象，系统将不会创建该 PendingIntent 对象而是直接返回 null。
  • FLAG_CANCEL_CURRENT: 如果当前系统中已经存在一个相同的 PendingIntent 对象，那么就将先将已有的 PendingIntent 取消，然后重新生成一个 PendingIntent对象。当你仅仅是改变intent中extra中的数据的时候，可以用这种方式获取新的PedingIntent对象
  • FLAG_UPDATE_CURRENT: 如果系统中已存在该 PendingIntent 对象，那么系统将保留该 PendingIntent 对象，但是会使用新的 Intent 来更新之前 PendingIntent 中的 Intent 对象数据，例如更新 Intent 中的 Extras。这个是经常使用的，因为Intent有更新的时候需要用到这个flag去更新你的描述(确切的说是更新extra data)，否则组件在下次事件发生或时间到达的时候extras永远是第一次Intent的extras，使用FLAG_CANCEL_CURRENT也能做到更新extras，只不过是先把前面的extras清除，另外FLAG_CANCEL_CURRENT和FLAG_UPDATE_CURRENT的区别在于能否新new一个Intent，FLAG_CANCEL_CURRENT能够新new一个Intent，而FLAG_UPDATE_CURRENT则不能，只能使用第一次的Intent。

如何判断PendingIntent相同

A.用不同的intent（相同的intent指的是：same operation, same Intent action, data, categories, and components,）特别注意更改”extra” contents没用。

B.用不同的requestCode。就是getActivity(Context, int, Intent, int)的第二个参数。

PendingIntent与Intent的区别

a. Intent是立即使用的，而PendingIntent可以等到事件发生后触发，PendingIntent可以cancel
b. Intent在程序结束后即终止，而PendingIntent在程序结束后依然有效,因为PendingIntent是系统持有这个对象
c. PendingIntent自带Context，而Intent需要在某个Context内运行
d. Intent在原task中运行，PendingIntent在新的task中运行

文档的部分说明

一个 PendingIntent 就是一个 Android 系统管理并持有的用于描述和获取原始数据的对象的标志 (引用)。这就意味着，即便创建该 PendingIntent 对象的进程被杀死了，这个 PendingItent 对象自己在其他进程中还是可用的。如果创建该 PendingIntent 对象的进程随后又重新获取了一个同类型的 PendingIntent（对于程序来讲，就是通过同样的方法获取的，例如都是通过 getActivity、getBroadcast、getService 方法来获取的，并且传递给 getXXX 方法的 Intent 对象的 Action 是相同的，Data 也是相同的，Categories 也是相同的，Components 也是相同的，Flags 也是相同的），如果之前获取的 PendingIntent 对象还有效的话，那么该进程获取到的 PendingItent 对象将获得同一个对象的引用，而且可以通过 cancel() 方法来从系统中移除它。

小结

当你把PendingIntent递交给别的程序进行处理时,PendingIntent仍然拥有PendingIntent原程序所拥有的权限(with the same permissions and identity).当你从系统取得一个PendingIntent时，一定要非常小心才行。比如，通常，如果Intent目的地是你自己的component（Activity/Service/BroadcastReceiver）的话，你最好采用在Intent中显示指定目的component名字的方式，以确保Intent最终能发到目的，否则Intent最后可能不知道发到哪里了。一个PendingIntent就是Android系统中的一个token（节点，这个应该是Linux或C\C++用语）的一个对象引用，它描述了一些将用于retrieve的数据（这里，这些数据描述了Intent及其最终的行为）。

这就意味着即使PendingIntent原进程结束了的话, PendingIntent本身仍然还存在，可在其他进程（PendingIntent被递交到的其他程序）中继续使用.如果我在从系统中提取一个PendingIntent的，而系统中有一个和你描述的PendingIntent对等的PendingInent, 那么系统会直接返回和该PendingIntent其实是同一token的PendingIntent，而不是一个新的token和PendingIntent。然而你在从提取PendingIntent时，通过FLAG_CANCEL_CURRENT参数，让这个老PendingIntent的先cancel()掉，这样得到的pendingInten和其token的就是新的了。

通过FLAG_UPDATE_CURRENT参数的话，可以让新的Intent会更新之前PendingIntent中的Intent对象数据，例如更新Intent中的Extras。另外，我们也可以在PendingIntent的原进程中调用PendingIntent的cancel()把其从系统中移除掉。

踢人踢设备

需求:
一. 账户不能同时在两个不同设备上登录
二. app只能在一台手机和一台pad上使用

踢人踢设备都是在登录的时候发生的,后台根据登录的时候传的参数进行判断返回数据,所以先看看登录的时候要都提交的字段,那些基本的数据就不提了.

  1. appname 应用的简化名字+平台号
    应用的简化名字 就是用来标识哪个应用, 可能公司有很多个不同的app, 这个名字就是用来标识不同的app的
    平台号 就是标识是哪种设备, 比如 手机版是1, pad是9

  2. mid 是设备的的唯一标识, 这个必须是唯一, 用来标识不同的设备, 先读取deviceid, 如果失败, 再读取androidid, deviceid -> androidId-> mac

踢人

在后台开启一个子线程, 每隔2分钟访问一下检测的接口, 看是否这个账号登录了, 如果已经登录了, 那么就会发送一个广播, 静态广播接受者接收到发送的广播后, 会执行登出的操作. 然后提示跳转到登录界面, 提示用户多地登录了, 同时杀死这个检测的子线程

踢设备

在登录的时候, 获取接口返回的code值, 判断是否是多设备登录, 如果是多设备登录, 提示用户弹出对话框, 提示用户注销掉之前的设备信息并且提交给改变设备的接口. 清除了的设备就不能再登录了, 如果再次在清除的设备上登录, 记得提示用户, “当前设备已经被禁用了, 无法登录”.

总结

这个踢人踢设备不是很难, 总的来说就是访问接口, 根据接口返回的不同code, 判断是哪种状态, 然后做相应的处理. 这个appname和mid这两个参数很重要, 后台就是根据这2个参数进行判断的.

SoundPool使用

MediaPlayer, VideoView, SoundPool的区别

Android中可以播放音频的有MediaPlayer, VideoView, SoundPool
他们的特点分别是:

MediaPlayer: 资源占用多, 延迟较长, 不支持多个音频同时播放, 一般用于播放比较大的音频文件

VideoView: 只能播放标准的MP4、3GP文件，其它格式的文件将无法播放

SoundPool: 内存占用少, 延时少, 可以播放多个音频

所以如果要播放短促、密集、延迟程度小的音频就要选择SoundPool

当调用loop()方法之后,这个Loopper线程就真正的开始工作了,不断的从自己的MessageQueue中取出Message,然后交给Meessage中的target调用dispatchMessage处理消息.

发送消息之后,Looper轮寻,获取msg之后,他是怎么有条不紊的交个对应的handler处理呢? 请注意上面

执行Looper.loop()之后,会让当前线程无限循环，不断从MessageQueue的实例中获取消息，然后回调msg.target.dispatchMessage(msg)方法。

那么dispatchMessage(msg)是个什么鬼.

这个方法就是分发给哪个handler去处理消息.但是处理消息的方式有2种,因为消息有2种封装方式.

具体哪2种呢:
一种的DataMessage,也就是Message,另一种是CallbackMessage,也就是Runnable对象.但是MessageQueue中只支持DataMessage,如果传入Runnable呢,那会怎么处理呢?

这里可以知道,如果是Runnable对象,首先会把这个对象转换成Message,然后添加到MessageQueue.
public final boolean postAtTime(Runnable r, long uptimeMillis)
public final boolean postDelayed(Runnable r, long delayMillis)
等这些传入Runnable对象都会做这样的转换.

再继续看上面的public void dispatchMessage(Message msg)方法,如果有回调就执行回调中的方法(就是Runnable中的run方法),如果只重写了handleMessage(),就执行handleMessage(),如果2者都有,还是只会执行回调中的方法.

handler2通过post(runnalbe),设置一个Runnable对象,首先把这个Runnable对象,转变成Message,然后发送得到Looper的MessageQueue中,

Looper轮寻到消息msg2,就交给msg2的target执行dispatchMessage方法,也就是回调runnable中的run方法.

Android中的Looper的使用

什么是Looper

Looper用来把一个线程封装成Looper线程,因为新new出来的线程不是Looper线程.
那么Looper线程有什么用呢,简单的讲,Looper线程可以接收其他线程发送的消息,并进行处理.

在打造Looper线程之前,先学习一下Looper的API,主要有一下几个方法

  • public static void prepare() 为当前线程提供一个Looper

  • public static void loop() 在线程中执行消息队列，在结束时通过调用looper.quit()

  • public Thread getThread() 返回和looper相关联的线程

  • public static Looper getMainLooper() 返回当前应用程序中依赖主线程的looper

  • public static Looper myLooper() 返回和当前线程关联的looper，如果没有looper与当前线程关联，则返回null

  • public static MessageQueue myQueue() 返回和当前线程相关联的消息队列，使用这个方法，必须确保当前线程与looper相关联

  • public static void prepareMainLooper() 初始化与主线程相关联的looper

Looper的工作原理

看图说话:

这里简单的描述一下 Looper线程中维护了唯一一个Looper对象,Looper对象中有一个MessageQueue(消息队列),这个是一个存放Message的队列.当有消息来的时候,就添加到MessageQueue中去.而Message是Handler发送的,这个Message对象中有由哪个handler发送它的引用,假如 MS1是Handler1发送的那么MS1中的target就引用Handler1.

当调用loop()方法之后,这个Loopper线程就真正的开始工作了,不断的从自己的MessageQueue中取出Message,然后处理

从上图中,我们可以发现 Looper Thread中只有一个Looper,可以有多个Handler.

什么情况下使用 Looper 线程

如果我们需要新建一个线程，并且这个线程要能够循环处理其他线程发来的消息事件，或者需要长期与其他线程进行复杂的交互，这时就需要用到Looper来给线程建立消息队列。

总结

Android的Looper类使用的5个要点:

  1. Looper类用来为一个线程开启一个消息循环。
    默认情况下android中新诞生的线程是没有开启消息循环的。（主线程除外，主线程系统会自动为其创建Looper对象，开启消息循环。）
    Looper对象通过MessageQueue来存放消息。一个线程只能有一个Looper，对应一个MessageQueue。

  2. 通常是通过Handler对象来与Looper进行交互的。Handler可看做是Looper的一个接口，用来向指定的Looper发送消息及定义处理方法。
    默认情况下Handler会与其被定义时所在线程的Looper绑定，比如，Handler在主线程中定义，那么它是与主线程的Looper绑定。
    mainHandler = new Handler() 等价于new Handler（Looper.myLooper()）.
    Looper.myLooper()：获取当前进程的looper对象，类似的 Looper.getMainLooper() 用于获取主线程的Looper对象。

  3. 在非主线程中直接new Handler() 会报如下的错误:
    E/AndroidRuntime( 6173): Uncaught handler: thread Thread-8 exiting due to uncaught exception
    E/AndroidRuntime( 6173): java.lang.RuntimeException: Can’t create handler inside thread that has not called Looper.prepare()
    原因是非主线程中默认没有创建Looper对象，需要先调用Looper.prepare()启用Looper。

  4. Looper.loop(); 让Looper开始工作，从消息队列里取消息，处理消息。

    注意：写在Looper.loop()之后的代码不会被执行，这个函数内部应该是一个循环，当调用mHandler.getLooper().quit()后，loop才会中止，其后的代码才能得以运行。

  5. 基于以上知识，可实现主线程给子线程（非主线程）发送消息。
